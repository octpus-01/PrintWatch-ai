## 1. 项目概述

### 1.1 背景与挑战
3D打印过程中常见的失败模式（如翘曲、层移、挤出不足、 spaghetti 怪物等）往往导致材料浪费和时间损耗。传统的人工监控效率低下，而基于传统计算机视觉的方法在面对复杂多变的打印环境时泛化能力较差。

### 1.2 项目目标
本项目旨在构建一套端到端的智能监控系统，通过以下核心能力解决上述痛点：
*   **智能识别**：利用强化学习（RL）自适应调整识别策略，实现多类型打印缺陷的精准分类。
*   **端云协同**：在高性能GPU端利用CUDA进行高效训练与微调，在资源受限的树莓派端实现实时推理。
*   **可视化监控**：提供直观的Web界面，实时展示打印状态与缺陷告警。

---

## 2. 系统整体架构

本系统采用“训练-部署-监控”分离的架构设计，主要包含三个核心模块：

1.  **训练端 (Training Node)**：基于NVIDIA GPU，负责数据集处理、模型预训练、强化学习策略优化及CUDA算子加速。
2.  **推理端 (Inference Node)**：基于Raspberry Pi，运行轻量化模型，通过摄像头采集图像并进行实时缺陷推断。
3.  **交互端 (Client UI)**：基于Web的可视化界面，接收推理端上报的数据并展示。

## 3. 核心算法：强化学习与多类型缺陷分类

### 3.1 缺陷类型定义
本项目针对3D打印过程中的典型故障进行了多分类建模，涵盖以下类别：
*   **正常 (Normal)**
*   **翘曲 (Warping)**：模型边缘脱离打印平台。
*   **层移 (Layer Shift)**：打印层发生错位。
*   **挤出异常 (Extrusion Issues)**：包括挤出不足或过度挤出。
*   ** spaghetti 故障**：丝材乱堆导致的打印失败。

### 3.2 强化学习识别策略
为了应对打印环境光照变化及模型几何复杂度的干扰，本项目引入了强化学习机制：
*   **状态空间 (State)**：当前帧图像特征、历史推理置信度、打印G-code上下文。
*   **动作空间 (Action)**：动态调整图像预处理参数（如曝光、对比度）、触发重识别机制、调整推理阈值。
*   **奖励函数 (Reward)**：基于分类准确率与误报率的加权反馈，引导Agent学习最优的识别策略。

## 4. 训练端优化：CUDA加速与模型微调

### 4.1 数据集准备与增强
*   **数据来源**：开源3D打印失败数据集 + 自采数据。
*   **预处理**：统一分辨率裁剪、去噪、数据标准化。

### 4.2 CUDA 训练优化
为了缩短模型迭代周期，训练端进行了深度的CUDA优化：
*   **算子融合**：自定义CUDA Kernel，将常用的图像预处理算子融合，减少显存读写开销。
*   **混合精度训练**：利用Tensor Cores支持FP16混合精度，在保持精度的同时提升吞吐量。
*   **数据加载流水线**：优化DataLoader，利用多进程与 pinned memory 实现CPU与GPU的数据传输重叠。

### 4.3 模型微调策略
采用迁移学习策略，基于预训练骨干网络（如ResNet/EfficientNet）进行微调：
*   **冻结骨干层**：保留通用特征提取能力。
*   **分类头重构**：针对多类型缺陷分类任务，重新设计全连接层。
*   **损失函数**：使用交叉熵损失函数（Cross-Entropy Loss）并结合Focal Loss解决样本不平衡问题。

## 5. 推理端优化：树莓派边缘部署

### 5.1 模型轻量化
为了在树莓派（ARM架构）上实现流畅运行，采取了以下压缩策略：
*   **模型剪枝**：剔除冗余的通道与神经元。
*   **量化感知训练 (QAT)**：将模型权重从FP32量化为INT8，大幅降低模型体积并加速整数运算。

### 5.2 推理引擎加速
*   **推理后端**：使用针对ARM优化的推理引擎（如TFLite / ONNX Runtime / NCNN）。
*   **多线程推理**：利用树莓派的多核CPU特性，设置合理的线程数进行并行推理。
*   **零拷贝优化**：在摄像头采集与推理输入之间使用共享内存，减少数据搬运延迟。

## 8. 快速开始与部署指南

### 8.1 环境要求
*   **训练端**：Ubuntu 20.04+, NVIDIA GPU (Compute Capability >= 6.0), CUDA 11.x+
*   **推理端**：Raspberry Pi 4B/5, Raspberry Pi OS (64-bit)

### 8.2 训练端启动
```bash
# 克隆仓库
git clone <your-repo-url>
cd <repo-name>/training

# 安装依赖
pip install -r requirements.txt

# 启动CUDA训练
python train.py --config config/default.yaml
```

### 8.3 推理端部署
```bash
# 在树莓派上安装依赖
sudo apt-get update
pip install -r requirements_edge.txt

# 运行推理服务
python edge_inference.py --model_path models/quantized_model.tflite
```

## 9. 常见问题 (FAQ)

**Q: 树莓派推理延迟过高怎么办？**
A: 请检查是否开启了INT8量化，并尝试减少摄像头的输入分辨率。同时确保没有其他高负载进程占用CPU。

**Q: 如何添加新的缺陷类别？**
A: 在数据集文件夹中新增对应类别的图片，修改配置文件中的 `num_classes`，并重新运行微调脚本。
